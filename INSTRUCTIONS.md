# JCS Abuja – Student Project Showcase Instructions

This document explains how to use and maintain this repository.

---

## 1. Purpose
This repository exists to showcase the **best student projects** from JCS Abuja’s web development programs.

It is:
- A public portfolio
- A credibility tool for JCS Abuja
- A motivation tool for students

---

## 2. How to Add a New Project

1. Ensure the student project:
   - Works correctly
   - Has a clear README in their own repository
   - Uses Git properly

2. Get student consent before featuring:
   - Full name OR first name only
   - GitHub username
   - Live demo link (if available)

3. Copy the project template from `README.md`

4. Paste it under **Featured Student Projects**

5. Fill in:
   - Project name
   - Student name
   - Cohort
   - Description
   - Features
   - Tech stack
   - GitHub repo link
   - Live demo link

---

## 3. Project Selection Guidelines

Only feature projects that demonstrate:
- Real problem-solving
- Clean code
- Proper folder structure
- Responsive UI
- Completion (not half-built apps)

Quality matters more than quantity.

---

## 4. Recommended Workflow

- Students keep projects in their own GitHub accounts
- You link or reference them here
- Optional: fork exceptional projects

---

## 5. Optional Enhancements

You may later:
- Convert this repo to GitHub Pages
- Add screenshots to projects
- Highlight “Project of the Month”
- Add alumni or cohort sections

---

## 6. Maintenance

- Review projects quarterly
- Remove broken demo links
- Update technologies as curriculum evolves
